1 Learning how to extract information, comprehend high level solution - When it comes to programming You'll be doing a lot more reading than writing most of the time.
- A tool / package / programming language can only be as smart as the person who uses it. The more you understand the better / optimized your solution.
- Most of the time the problem is generic enough that we find existing tools and solutions for it. Other times when there's no real guidance, it pays huge dividends to learn how to collect the available documentation to ground yourself, extract the relevant bits to the use case then start solving.
- Regardless of what solution you use - from others who already solved it, built it yourself, or from LLM suggestion - once you commit the code it is your responsibility to maintain it i.e. it will come back at you. The faster you learn how something works, even at abstract level, the better position you are when this comes.		 - Look back on project retros. Find an improvement point.
